## 0. 정체성 전환 (오늘의 가장 큰 결정)

**Rona = "AI 업무 도구" (실용·도구적).** "따뜻한 동행 펫"이 아니다.

브레인스토밍 중반까지 "같이 가요 🌿" 같은 동행 펫 톤으로 잡았으나, 사용자가 명시적으로 **AI 업무 도구** 정체성을 선택했다. 카피·디자인 방향 전체가 이 결정을 따른다.

| | 동행 펫 (버림) | AI 업무 도구 (채택) |
|---|---|---|
| 톤 | "같이/함께 가요 🌿" | "실습 진행 중" 실용적 |
| 진행 표현 | "5걸음째 같이 왔어요" | "2/5 완료" 명확·효율 |
| 핵심 가치 | 정서적 안심 | 업무 진척·생산성 |
| 마스코트 | 항상 크게(주인공) | 빈 상태에만, 진행 중엔 정보 우선 |

**단, 정체성이 바뀌어도 유지되는 원칙: 정직성(판단 안 함).** 도구는 오히려 정확성이 생명. "해결 중"(판단) 대신 "진행 중"(사실).

## 1. 코코가 보여줄 수 있는 것 (데이터 천장)

코코는 터미널/세션을 직접 못 본다. 아는 것은 두 가지뿐:

1. **서버 이벤트 8종** (`types.ts` SkillEventType) — "단계 저장됨", "도구 사용", "완주" 같은 거친 신호 + 시각.
2. **로컬 진행표** (`rona-progress.html`, `progress-reader.ts`) — 실습별 단계 서사(제목·state·what·detail) + 용어집(glossary) + 목표.

**볼 수 있음:** 실습 내용/단계 구성(=대본), 진행 위치, 용어 풀이.
**못 봄:** 대화 내용, 터미널 출력, 에러 메시지, 실시간(20초 폴링).

→ 코코의 역할은 "세션 감시"가 아니라 **"실습 대본을 아는 길잡이"**. 터미널이 못 주는 것(맥락·위치·용어)을 보조한다. 터미널과 중복되면 코코는 잉여다.

### 단계의 출처 (중요)
"단계"는 **실습마다 고유**하다(표준 템플릿 아님). 진짜 단계는 이미 Rona 스킬의 `SKILL.md "## 단계"` 섹션에 사람이 읽게 적혀 있다. **install 스킬이 이를 진행표(`rona-progress.html`)에 써넣는다는 전제** 위에서만 코코가 단계를 정직하게 보여줄 수 있다. (코코 레포 밖 의존성 — [[2026-06-04-coco-auto-companion-design]]의 install 전제와 같은 성격)

진행표가 없으면 → 서버 이벤트만으로는 "끝(분모)"을 모르므로 비율 진행바는 거짓이 된다. 그땐 "발자국"식 표현으로 폴백(별도).

---

## 2. 카피 가이드 (보이스 원칙) — 모든 문구의 기준

1. **AI 업무 도구 톤** — 실용·효율. 진행/완료를 명확히. (동행 펫의 "같이/걸음🌿" 톤 아님)
2. **판단 아닌 관찰 (정직성)** — Rona가 아는 것만 말한다. "잘 가고 있어요"·"해결 중"(품질·결과 판단) ✗ / "진행 중"·"2단계 완료"(사실) ✓. 막혀도 거짓이 되지 않는 문구만.
3. **입문자 눈높이** — "진행도/상세" 같은 사무어보다 쉬운 말. AI 용어는 풀어줌.
4. **짧게** — 라벨은 한 호흡. 자명한 정보엔 라벨을 붙이지 않는다.

이 가이드는 앞으로 새 문구를 추가할 때마다 적용한다.

## 3. 첫 화면 — 확정 사양

### 3.0 비주얼 톤 (마스코트 제거)
- **마스코트(코코 얼굴) 제거.** AI 업무 도구 정체성엔 캐릭터보다 정보·차분함이 맞다. 손그림 마스코트는 못생겨서 뺐다.
- **색**: 중성 회색 베이스(--ink/line) + 코랄(#f0635a)은 **포인트만**(현재 단계·진행바·펄스·로고 점). 코랄 남발 금지.
- **헤더**: 핑크 배경박스 없음. 작은 코랄 점 로고 + 텍스트 + 톱니.
- Linear/Notion 같은 차분한 도구 톤. 여백 넉넉, 그림자 절제, 테두리 얇게.

### 3.1 빈 상태 (시작 전, 추적 중인 실습 없음)
```
[헤더]   ● Rona  ⚙           ← 코랄 점 로고 + 텍스트
[오브]   (궤도 오브: 얇은 링 + 도는 점 + 중심점, ~46px)  ← 마스코트 대신 추상 비주얼
[메인]   AI로 오늘의 업무를 해결해볼까요?
[보조]   터미널에서 Rona 실습을 시작하면 이 창에 자동으로 나타나요
[대기]   ● 실습을 기다리는 중
[폴백]   안 보이나요? 폴더 직접 추가 →    ← 작게(자동 동행 폴백)
```
- **궤도 오브**: 마스코트 대신 빈 상태를 채우는 추상 비주얼. 얇은 링을 코랄 점이 도는 위성 형태. "대기 중·살아있음"을 미니멀하게.
- "해결해볼까요?"는 **시작 권유**라 빈 상태에만 맞다(아직 시작 전이므로 "해결"이 판단이 아님 — 권유). 진행 중 화면엔 쓰지 않는다.
- **자동 동행과 일관**: 메인은 "시작하면 자동으로 나타나요"(자동 대기), 수동 폴더 추가는 작은 폴백 링크로. ([[2026-06-04-coco-auto-companion-design]]) "실습 폴더 추가" 큰 버튼은 자동 동행 설계와 모순이라 제거됨.

### 3.2 진행 중 (추적 중인 실습 있음) — 개념적 1:1
```
[헤더]    스터디 마케팅 자료 정리        ⚙   ← 실습명을 헤더에
[상태]    ● 실습 진행 중            ← 펄스 점(깜빡임)
[진행바]  ▓▓▓▓░░░░░  (그라데이션 + shine 애니메이션)
          2 / 5 완료                    ← 라벨 없이 숫자
[단계]    3칸 창:
            ✓ 자료 주제별 나누기   [완료]   (흐림 .42)
            ● skill 단위 결정하기  [진행 중] (또렷, 코랄 강조)  ← 현재
            다음
              문서 템플릿 작성             (흐림 .6)
[용어]    🤖 AI 용어 check!
            skill — 반복 업무를 묶은 단위
            knowledge-base — 주제별 지식 폴더   (한 줄 압축, 펼침)
```

### 확정된 UI/UX 결정 목록
| 요소 | 결정 | 근거 |
|---|---|---|
| 정체성 | AI 업무 도구 | 사용자 선택 |
| 헤더 | 실습명 직접 표시 | 맥락이 먼저 |
| 상태 라벨 | "● 실습 진행 중" + 펄스 점 | 생동감(도구가 일하는 중) — "해결" 아닌 "진행"(정직) |
| 진행바 | 그라데이션(노랑→주황→코랄) + shine | 도구 톤, 밋밋함 탈출 |
| 진행 숫자 | "2 / 5 완료" (라벨 생략) | 자명한 정보엔 라벨 불필요 |
| 단계 표시 | **3칸 창** (직전·현재·다음) | 글 폭탄 방지. 끝은 진행바/숫자로 앎 |
| 단계 위계 | 완료 흐림(.42) / 현재 또렷+코랄 / 다음 흐림(.6) / 그 이후 숨김 | 시선을 "지금"에 집중 |
| 끝 처리 | 첫 단계=현재+다음2 / 마지막=직전2+현재("마지막!") | 항상 3칸 유지 |
| 단계 칩 | "완료 / 진행 중" | 표준·정확(도구 톤) |
| 단계 섹션 라벨 | 없음 | 현재 단계가 코랄로 또렷해 불필요 |
| 용어집 | "🤖 AI 용어 check!" — 한 줄 압축("용어 — 뜻"), 항상 펼침 | 3칸 창으로 공간 확보됨. 펼쳐도 안 길어. 입문자가 "누를 생각 못 하는" 문제 해소 |
| 실습 1개 | 리스트 숨김(개념적 1:1) / 여럿이면 전환 리스트 | 입문자는 한 실습에 집중 |

### 3칸 창 규칙
항상 3칸. 현재를 가운데 두고 앞뒤로 채운다.
- 보통: 직전 1 + **현재** + 다음 1
- 첫 단계: **현재** + 다음 2 (직전 없음)
- 마지막: 직전 2 + **현재** (다음 없음, "마지막!" 칩 + 완주 예고)

진행하면 이 3칸 창이 미끄러지듯 따라 이동. 전체 단계 수는 진행바·숫자(2/5)로 파악.

## 4. 진행도 계산 주의 (구현 시)

`panel.ts`의 `stepCounts`는 `state==="done"` 개수로 진행도를 센다. active는 안 셈. 그래서 "지금 3번째 진행 중"이어도 완료는 2개 → 진행바는 **2/5**로 뜬다. "지금 N번째"와 "M/total 완료"는 다른 숫자이며 둘 다 사실. 카피에서 혼동하지 말 것. (진행바 = 완료 기준)

## 5. 구현 범위 (후속 — 이 문서는 계획만)

코코 레포에서 손댈 것:
- `src/renderer/panel/panel.ts` — `richDetail`을 간결 뷰로(3칸 창, 위계, 용어 한 줄, 도구 톤 카피). 진행도 헬퍼.
- `src/renderer/renderer.ts` — (용어집 항상 펼침이면 토글 불필요)
- `src/renderer/index.html` (CSS) — 그라데이션 진행바 + shine, 펄스 점, 단계 위계(opacity), 한 줄 용어.
- 빈 상태 카피 교체.

install 스킬 쪽 전제 (코코 밖):
- 진행표(`rona-progress.html`)에 **실습별 고유 단계**를 써넣어야 함. ([[2026-06-04-coco-auto-companion-design]] 참조)

검증 자산:
- `handoff/mockups/sample-skill/` — 실제 Rona 스킬(rona-fee45e31)의 진짜 단계로 만든 진행표 + 마커 샘플. 코코로 직접 렌더 검증 가능.

## 6. 범위 밖
- 마스코트 애니메이션 강화(coco.ts mood) — 도구 톤이라 절제.
- 진행표 없을 때 "발자국" 폴백 — 별도.
- 막힘 도움/물어보기 버튼 — 보류(코코가 먼저 들이대는 건 침습적이라 기각). 사용자 능동 호출 버튼은 후속 검토.
